3. Zone Separation

Spec requires

  • Cloud zone (top 1/3) and river zone (bottom 2/3) must be visibly distinct
  • Soft gradient boundary between them — not a hard line, but you can feel the difference
  • River zone is slightly warmer/darker than cloud
  • "Valleys of light" between blobs — gaps glow warmer than areas under blobs
  • Flow streaks only in river zone, giving it motion and life

What was built

  • Zones are invisible — just a uniform dark void
  • The "gradient boundary" is 3% opacity amber — literally imperceptible
  • No warmth difference between zones
  • No "valleys of light" — everything is the same darkness
  • Flow streaks are also 3% opacity — might as well not exist

4. Fixed Tasks (Rocks)

Spec requires

  • Rounded hexagons or rounded rectangles with subtle bevel/shadow
  • Geometric where blobs are organic — clear visual contrast
  • Neutral stone palette — warm grays, tans, slate
  • Flow streaks PART AROUND them (the key "river around rocks" metaphor)

What was built

  • Tiny gray rounded rectangles — correct shape but way too small
  • Appear in wrong zones (positioning bug)
  • No bevel, no shadow, no presence
  • Flow streaks don't interact with them at all

Summary: The spec is detailed and good. The implementation ignored most of it.

The subagent that built the viewer wrote structurally correct code (SSE works, drag-drop works, click panel works) but got the visual design completely wrong. Everything that makes this feel like a river — zones, warmth, flow, organic shapes, the breathing quality — was either skipped or done at such low opacity it's invisible.

The fix: Rebuild viewer/river.js following the spec faithfully. The backend, SSE, and HTTP server are all solid — only the Canvas rendering needs to be redone.
